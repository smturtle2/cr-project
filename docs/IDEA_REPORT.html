Diffusion의 약점을 찌른다: EMRDM은 multi-step sampling (~3초), DiffCR도 ~1초. Transformer single forward pass(~0.1초)로 동등한 성능을 달성하면 "비슷한 성능, 10~30배 빠름"이라는 강력한 기여가 성립.

1. 제안 아키텍처 개요

SAR (2ch) ─→ [SAR Encoder] ──────────┐ │ ▼ ┌──────────────┐ Optical (13ch) ─→ [Opt Encoder] ─→│ Cross-Modal │──→ [Transformer Blocks] ──→ [Decoder] ──→ +Skip ──→ Output (13ch) │ Transformer │ ▲ ▲ │ Attention │ │ │ └──────────────┘ Frequency-Aware Cloud-Adaptive 모듈 ① Attention (모듈 ②) Modulation (모듈 ③) │ ┌─────────┴─────────┐ │ Spatial Branch │ │ (Window/Linear Attn)│ │ Frequency Branch │ │ (FFT Global Attn) │ └───────────────────┘
3가지 핵심 모듈:
① SAR-Optical Cross-Modal Transformer — SAR의 지표 구조를 Optical 복원에 직접 주입
② Frequency-Aware Attention — 구름의 주파수 특성을 활용한 공간+주파수 이중 어텐션
③ Cloud-Adaptive Feature Modulation — 구름 두께에 따라 SAR/Optical 의존도 동적 조절
ACA-CRNet 대비 변경점:
• Head Conv(single) → Dual-Branch Encoder (SAR/Optical 분리)
• ResBlock(Conv) → Transformer Block (Window/Linear Attention + FFN)
• AC-Attention(self-similarity) → Cross-Modal Attention (SAR↔Optical 상호작용)
• 유지: Global Residual Skip, CARL Loss, 학습 설정

전체 아키텍처 구체적 설명

제안 모델은 ACA-CRNet의 단일 CNN 파이프라인을 Dual-Branch Encoder + Transformer Backbone + Preconditioned Decoder로 업그레이드한다. 전체 데이터 흐름은 다음과 같다:

  1. 입력: 구름 낀 Optical 이미지(13채널, Sentinel-2 전 밴드)와 동일 지역의 SAR 이미지(2채널, Sentinel-1 VV/VH 편파)가 동시에 입력된다.
  2. Dual-Branch Encoding: SAR과 Optical을 독립적인 인코더로 처리하여 각 모달리티의 고유한 특성을 보존한 feature map을 추출한다. ACA-CRNet의 단순 concat과 달리, 모달리티별 특화된 표현을 학습한다.
  3. Cross-Modal Fusion (모듈 ①): SAR feature를 Key/Value로, Optical feature를 Query로 사용하는 cross-attention을 통해 SAR의 지표 구조 정보를 Optical 복원에 주입한다.
  4. Transformer Processing (모듈 ②): 융합된 feature를 4-level U-Net 구조의 Transformer로 처리한다. 각 블록에서 Linear Attention(전역), Window Attention(로컬), FFT Attention(주파수)의 3중 처리를 수행하여, Diffusion의 iterative refinement를 단일 pass로 대체한다.
  5. Cloud-Adaptive Modulation (모듈 ③): 각 Transformer 블록의 출력을 구름 밀도에 따라 적응적으로 변조하여, 영역별 최적의 복원 전략을 적용한다.
  6. Preconditioned Output (섹션 6): EMRDM에서 차용한 preconditioning으로 구름 밀도에 따라 skip connection 비율과 출력 스케일을 조절한다.
  7. Global Residual: 최종 출력에 입력 Optical 이미지를 더하는 global skip connection을 유지하여, 네트워크는 잔차(clean - cloudy)만 학습한다.

예상 모델 규모: ~15M 파라미터, ~40G FLOPs (256x256), 추론 ~0.1초/이미지 (RTX 3090 기준)

2. 모듈 ① — SAR-Optical Cross-Modal Transformer

SAR Cross-Attention   ACA-CRNet의 단순 concat을 대체하는 핵심 기여.

기존 문제: ACA-CRNet은 SAR과 Optical을 concat 후 같은 Conv로 처리 → SAR 고유 정보(지표 구조, 구름 투과) 활용 부족.
제안: SAR feature를 Key/Value, Optical feature를 Query로 사용하는 cross-attention → SAR의 지표 정보가 Optical 복원을 직접 가이드.

설계 의사코드

# Cross-Modal Transformer Attention
class CrossModalAttention(nn.Module):
    def forward(self, opt_feat, sar_feat):
        Q = self.q_proj(opt_feat)          # Optical → Query
        K = self.k_proj(sar_feat)          # SAR → Key
        V = self.v_proj(sar_feat)          # SAR → Value

        # SAR 구조 정보로 Optical 복원 가이드
        attn = softmax(Q @ K.T / sqrt(d))
        out_opt = attn @ V + opt_feat      # SAR-guided Optical feature

        # (선택) 역방향: Optical → SAR 상호작용 (mutual injection)
        Q2 = self.q2_proj(sar_feat)
        K2 = self.k2_proj(opt_feat)
        V2 = self.v2_proj(opt_feat)
        out_sar = softmax(Q2 @ K2.T / sqrt(d)) @ V2 + sar_feat

        return out_opt, out_sar

구체적 설명: SAR-Optical Cross-Modal Transformer

핵심 원리: SAR(합성 개구 레이더)는 마이크로파를 사용하므로 구름을 투과하여 지표면의 구조적 정보(건물 윤곽, 도로, 지형 등)를 획득할 수 있다. 반면 Optical 이미지는 구름에 의해 지표 정보가 완전히 가려진다. Cross-Modal Transformer는 이 두 모달리티의 상보적 특성을 최대한 활용한다.

  • 단계 1 — 독립 인코딩: SAR(2채널)과 Optical(13채널)을 각각의 전용 인코더로 처리하여 모달리티 고유의 feature map을 추출한다. SAR 인코더는 스펙클 노이즈에 강건한 feature를, Optical 인코더는 구름 아래 잔존하는 스펙트럼 정보를 추출한다.
  • 단계 2 — Cross-Attention 융합: Optical feature를 Query로, SAR feature를 Key/Value로 사용하는 cross-attention을 수행한다. 이를 통해 "Optical의 어떤 위치가 SAR의 어떤 구조 정보를 참조해야 하는가"를 어텐션 가중치로 학습한다. 구름이 두꺼운 영역은 SAR에 대한 어텐션 가중치가 높아지고, 맑은 영역은 자기 자신(Optical)에 대한 의존도가 유지된다.
  • 단계 3 — (선택) 양방향 상호작용: 역방향으로 SAR도 Optical의 스펙트럼 정보를 참조하여, SAR feature의 색상/텍스처 표현력을 보강한다. 이 양방향 주입(Mutual Injection)은 단방향 대비 평균 0.3~0.5 dB의 추가 개선을 보인다 (W-Shaped Fusion 참조).
  • 효율성: SAR 해상도를 1/4로 다운샘플하여 Key/Value를 생성하면 cross-attention 연산량을 75% 절감하면서도 구조 정보 전달에는 충분하다.

3. 모듈 ② — Frequency-Aware Attention

Frequency Transformer   구름의 주파수 특성을 활용한 이중 도메인 어텐션.

물리적 근거: 두꺼운 구름 = 저주파 성분 지배, 얇은 구름 = 중~고주파 혼재, 지표 텍스처 = 고주파. 공간 도메인만으로는 구름/지표 분리가 어렵지만, 주파수 도메인에서는 구름과 지표의 신호가 자연스럽게 분리된다.

설계 의사코드

# Frequency-Aware Dual-Branch Attention
class FreqAwareAttention(nn.Module):
    def forward(self, x):
        # Spatial Branch — 로컬 텍스처
        spatial_out = self.window_attention(x)         # Window/Linear Attention

        # Frequency Branch — 글로벌 구조 + 구름 분리
        x_freq = torch.fft.rfft2(x, norm='ortho')     # → 주파수 도메인
        x_freq = self.freq_attention(x_freq)           # 주파수 도메인 attention
        freq_out = torch.fft.irfft2(x_freq, norm='ortho')  # → 공간 복원

        # Adaptive Fusion
        gate = self.gate(torch.cat([spatial_out, freq_out], dim=1))
        return gate * spatial_out + (1 - gate) * freq_out

구체적 설명: Frequency-Aware Attention

핵심 원리: 구름은 이미지의 주파수 스펙트럼에서 뚜렷한 특성을 보인다. 두꺼운 구름은 넓은 면적을 균일하게 덮으므로 저주파 성분이 지배적이고, 지표면의 건물/도로/식생 텍스처는 고주파 성분에 집중된다. 이 물리적 차이를 활용하면, 주파수 도메인에서 구름 신호와 지표 신호를 효과적으로 분리할 수 있다.

  • Spatial Branch (로컬 텍스처 복원): Window Attention 또는 Linear Attention을 사용하여 공간 도메인에서 로컬 텍스처 패턴을 복원한다. 구름 경계의 날카로운 전환, 작은 구조물의 디테일 등 고해상도 공간 정보를 담당한다. Window 크기는 8x8로 설정하여 로컬 패치 내 self-attention을 수행한다.
  • Frequency Branch (글로벌 구조 복원): 2D FFT로 feature를 주파수 도메인으로 변환한 후, 주파수 공간에서 attention을 수행한다. FFT의 각 주파수 bin은 전체 이미지에 대한 글로벌 정보를 담고 있으므로, O(n log n) 복잡도로 전역 수용 영역(global receptive field)을 확보한다. 주파수 도메인에서 구름의 저주파 성분을 억제하고 지표의 고주파 성분을 보존/복원하는 필터링을 학습한다.
  • Adaptive Gate Fusion: 두 브랜치의 출력을 시그모이드 게이트로 적응적으로 결합한다. 구름이 두꺼운 영역에서는 Frequency Branch(글로벌 구조 정보)에 더 높은 가중치를 부여하고, 맑은 영역이나 디테일이 중요한 부분에서는 Spatial Branch에 가중치를 부여한다.
  • 성능 기대: SFHformer에서 이 이중 브랜치 구조가 31개 데이터셋에서 SOTA를 달성했으며, FFT 브랜치 추가만으로 최대 2.12 dB 개선을 보임. 구름 제거 태스크에서는 구름의 주파수 특성이 명확하므로 더 큰 개선이 기대된다.

4. 모듈 ③ — Cloud-Adaptive Feature Modulation

SAR Degradation-Aware   구름 두께별 적응적 처리.

문제: 두꺼운 구름(Optical 정보 0%) vs 얇은 구름(부분 투과) vs 맑은 영역을 동일하게 처리하면 비효율.
제안: SAR-Optical 유사도에서 구름 밀도를 추정하고, 이를 조건으로 Transformer feature를 변조.

설계 의사코드

# Cloud-Adaptive Feature Modulation
class CloudAdaptiveModulation(nn.Module):
    def __init__(self, channels):
        self.cloud_estimator = nn.Sequential(    # SAR-Optical 유사도 → 구름 밀도
            Conv2d(channels*2, channels//4, 1), GELU(),
            Conv2d(channels//4, 1, 1), Sigmoid())
        self.scale = nn.Linear(1, channels)      # 채널별 스케일
        self.shift = nn.Linear(1, channels)      # 채널별 시프트

    def forward(self, feat, sar_feat, opt_feat):
        # 구름 밀도 추정 (0=맑음, 1=두꺼운 구름)
        sim = F.cosine_similarity(sar_feat, opt_feat, dim=1, keepdim=True)
        cloud_density = self.cloud_estimator(torch.cat([sar_feat, opt_feat], dim=1))

        # Feature Modulation (AdaIN 스타일)
        scale = self.scale(cloud_density).unsqueeze(-1).unsqueeze(-1)
        shift = self.shift(cloud_density).unsqueeze(-1).unsqueeze(-1)
        return feat * (1 + scale) + shift    # 구름 밀도에 따라 feature 변조
직관: 맑은 영역 → scale≈0, shift≈0 (feature 유지), 두꺼운 구름 → scale/shift가 SAR 정보 기반으로 feature를 크게 변조하여 재구성.

구체적 설명: Cloud-Adaptive Feature Modulation

핵심 원리: 하나의 위성 이미지 안에서도 구름의 두께와 분포는 매우 불균일하다. 맑은 영역(구름 없음), 얇은 구름(반투명, 지표 일부 가시), 두꺼운 구름(지표 완전 차단)이 공존하며, 각 영역은 서로 다른 복원 전략이 필요하다. 이 모듈은 픽셀별로 구름 밀도를 추정하고, 그에 따라 Transformer feature를 적응적으로 변조한다.

  • 구름 밀도 추정: SAR feature와 Optical feature 간의 코사인 유사도를 기반으로 구름 밀도를 추정한다. 맑은 영역에서는 SAR과 Optical이 유사한 구조를 보이므로 유사도가 높고(=구름 밀도 낮음), 두꺼운 구름 영역에서는 Optical이 완전히 다른 패턴을 보이므로 유사도가 낮다(=구름 밀도 높음). 이 유사도를 경량 CNN(1x1 Conv 2층)에 통과시켜 0~1 범위의 구름 밀도 맵을 생성한다.
  • AdaIN 스타일 변조: 추정된 구름 밀도를 조건으로 채널별 scale(γ)과 shift(β)를 생성한다. 이는 StyleGAN의 Adaptive Instance Normalization과 유사한 메커니즘이다. output = feat * (1 + γ) + β로, 구름 밀도에 따라 feature의 각 채널을 스케일링하고 시프트한다.
  • 적응적 동작:
    - 맑은 영역 (density ≈ 0): scale ≈ 0, shift ≈ 0 → feature가 거의 변하지 않음 (원본 Optical 정보 보존)
    - 얇은 구름 (density ≈ 0.3~0.6): 중간 정도의 변조 → Optical의 잔존 정보와 SAR 정보를 블렌딩
    - 두꺼운 구름 (density ≈ 1): 강한 변조 → SAR 기반으로 feature를 거의 완전히 재구성
  • 적용 위치: 각 Transformer 블록의 출력에 적용하며, 블록마다 독립적인 modulation 파라미터를 학습한다. 이를 통해 초기 블록은 대략적인 구조 복원에, 후반 블록은 세밀한 텍스처 복원에 집중하는 계층적 적응이 가능하다.

구체적 설명: 효율적 Attention Backbone 설계

핵심 문제: 256x256 원격 탐사 이미지를 패치 크기 4로 토큰화하면 64x64 = 4,096개 토큰이 생성된다. Full Self-Attention의 O(n²) 복잡도는 이 규모에서도 병목이 되며, 특히 학습 시 배치 크기를 제한한다. 따라서 성능을 유지하면서 연산을 줄이는 효율적 attention 설계가 필수적이다.

  • Linear Attention (전역 문맥, O(n·d)): 커널 함수 φ를 사용하여 softmax(QK^T)V를 φ(Q)(φ(K)^T V)로 재배열한다. 이렇게 하면 n² 대신 n·d 복잡도로 전역 수용 영역을 확보한다. ACL 논문은 이 Linear Attention이 수학적으로 SSM(State Space Model)과 동치임을 증명했으며, 추가로 다중 스케일 dilated convolution으로 로컬 정보를 보강한다. 구름 제거에서는 멀리 떨어진 맑은 영역의 정보를 참조해야 하므로 전역 수용 영역이 핵심적이다.
  • Window Attention (로컬 텍스처, O(n·w²)): HAT 스타일의 8x8 윈도우 내 self-attention으로 로컬 텍스처 패턴을 정밀하게 모델링한다. Overlapping Cross-Attention(OCA)을 통해 윈도우 경계의 정보 단절 문제를 해결한다. 구름 경계 근처의 날카로운 전환이나 세밀한 지표 텍스처 복원에 효과적이다.
  • FFT Attention (주파수 도메인, O(n log n)): 모듈 ②의 Frequency Branch로, FFT의 O(n log n) 복잡도로 전역 주파수 패턴을 처리한다. 구름의 저주파 특성을 분리하는 데 특화된 경로이다.
  • 블록 구성: 각 Transformer 블록은 [Linear Attn → Window Attn → FFT Attn → FFN] 순서로 구성하며, U-Net 구조의 4-level 인코더-디코더에 배치한다. 레벨별 블록 수는 [2, 4, 6, 4]로 설정하여 bottleneck에서 더 깊은 처리를 수행한다.

6. Diffusion에서 빌려오되 Diffusion이 아닌 기법

Diffusion의 성능 비결을 분석하고, iterative sampling 없이 핵심 아이디어만 이식.

Diffusion의 강점Transformer로 이식하는 방법출처

Link
핵심 원칙: Diffusion에서 가져오는 것은 conditioning 기법과 loss 설계이지, iterative sampling은 가져오지 않는다. 이것이 "Diffusion 기반이 아닌" 아키텍처의 경계선이다.

Preconditioning 이식 설계

# EMRDM의 Preconditioning → Transformer에 이식
class PreconditionedForward(nn.Module):
    def forward(self, x, cloud_density):
        # cloud_density: Cloud-Adaptive Modulation에서 추정한 값
        c_in   = 1 / sqrt(cloud_density**2 + sigma_data**2)
        c_skip = sigma_data**2 / (cloud_density**2 + sigma_data**2)
        c_out  = cloud_density * sigma_data / sqrt(cloud_density**2 + sigma_data**2)

        x_scaled = c_in * x                     # 입력 정규화
        net_out = self.transformer(x_scaled)     # Transformer forward
        return c_skip * x + c_out * net_out      # 적응적 skip + 출력 스케일링

구체적 설명: Diffusion 기법의 비-반복적 이식

핵심 원리: Diffusion 모델(EMRDM, DiffCR)이 SOTA 성능을 달성하는 이유는 iterative sampling 자체가 아니라, 그 과정에서 활용되는 conditioning 기법과 loss 설계에 있다. 이 핵심 아이디어들을 Transformer의 single forward pass에 이식하면, 반복 추론 없이도 유사한 성능을 기대할 수 있다.

  • Preconditioning (EMRDM 차용): Diffusion 모델은 각 시간 스텝(노이즈 수준)에 맞게 입출력을 적응적으로 스케일링한다. 구름 제거에서는 "구름 밀도"가 Diffusion의 "노이즈 수준"에 대응한다. 모듈 ③에서 추정한 구름 밀도를 Diffusion의 σ(t)처럼 사용하여:
    - c_in: 입력 feature를 구름 밀도에 맞게 정규화 (큰 열화에 대해 수치 안정성 확보)
    - c_skip: 원본 입력을 얼마나 직접 전달할지 결정 (맑은 영역 → skip 비율 높음)
    - c_out: 네트워크 출력의 스케일 조절 (두꺼운 구름 → 큰 수정 필요)
    이 3개 스칼라는 구름 밀도의 해석적 함수로 계산되므로 학습 파라미터를 추가하지 않는다.
  • Residual Noise Prediction (Resfusion 차용): 기존 구름 제거 모델은 clean 이미지 자체를 예측하지만, Resfusion의 잔차 예측 방식을 차용하여 net(cloudy) = clean - cloudy (잔차)를 예측한다. 잔차 학습은 네트워크가 "변해야 할 부분"에만 집중하게 하므로, 맑은 영역에서의 불필요한 수정을 줄이고 학습을 안정화한다.
  • Direct Bridge (DB-CR 참조): DB-CR은 Gaussian noise를 경유하지 않고 cloudy→clean을 직접 매핑하는 bridge를 사용한다. 이는 본질적으로 Transformer가 하는 일과 동일하므로 별도 수정 없이, "Diffusion의 장점은 direct mapping으로도 달성 가능하다"는 논리적 근거로 활용한다.
